## 7. Desktop Performance — Additional Suggestions

Beyond the recovery guide steps:

1. **Move AI_Agent_Skills out of OneDrive.** The .git directory inside OneDrive causes lock contention on every git operation. Move to C:\Users\User\AI_Agent_Skills (outside OneDrive) and use git push/pull for sync instead.

2. **Reduce Bitdefender real-time scanning scope.** After getting admin back, add exclusions for: node_modules, .git, .vscode, AppData\Local\AnthropicClaude, VSCodespace.

3. **Switch to Windows Defender only.** Bitdefender is heavy (~700MB RAM). If you want lighter AV, switch to just Defender + configure its exclusions. Saves ~500MB RAM.

4. **Schedule Windows Search rebuilds.** After restricting scope (remove G:, H:, iCloudDrive, OneDrive from index), rebuild once and it'll be much lighter.

5. **Check for Windows Update stuck downloads.** Run `Get-WindowsUpdate` or check Settings → Windows Update. Stuck updates can cause background CPU drain.

6. **Docker Desktop WSL integration.** We disabled Docker startup, but WSL2 may still have a Docker distro consuming resources. Run `wsl --list --verbose` and `wsl --shutdown` if Docker distros are running.

## 8. Cold Storage Architecture

### Tier Design
```
HOT (milliseconds)     → Supabase pgvector, D1 edge, local SSD
WARM (seconds)         → Google Drive, OneDrive, DuckDB, CockroachDB, Neon
COLD (minutes)         → R2 archive, HDD H: drive, NordCloud
FROZEN (on-demand)     → iDrive historical backups, old PC data
```

### Auto-Archive Rules (implement in Phase 7)
- Files not accessed in 90 days → move from HOT to WARM
- Files not accessed in 180 days → move from WARM to COLD (R2 or H: drive)
- Files not accessed in 365 days → move to FROZEN (iDrive or NordCloud)
- Always keep: source code, configs, credentials, documents < 1 year old

### MacBook SSD Strategy
Since internal SSD space is always a concern:
- Keep only active projects on internal SSD
- SD card: overflow for media, downloads, VM images
- NordCloud (3TB): encrypted backup of everything on Mac
- Google Drive: shared documents (file streaming mode, not mirror)
- Archive anything > 6 months old to R2 or NordCloud
