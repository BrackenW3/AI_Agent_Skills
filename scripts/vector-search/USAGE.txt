VECTOR SEARCH INDEXING PIPELINE
================================

A production-quality Python pipeline for building vector search indexes
from files across OneDrive, Google Drive, and local directories.

FILES CREATED
=============

1. config.py (169 lines)
   - Configuration management with dataclasses
   - Loads from environment variables / .env files
   - AzureOpenAIConfig, SupabaseConfig, IndexingConfig
   - Global config getter with validation
   - Supports all major settings: API keys, endpoints, chunk sizes, rate limits

2. indexer.py (523 lines)
   - Main indexing pipeline orchestrator
   - DocumentChunker: Text chunking with configurable overlap (50% default)
   - FileProcessor: Multi-format text extraction (.txt, .md, .pdf, .docx, .py, .json, .csv, .html)
   - EmbeddingGenerator: Azure OpenAI async API with rate limiting and retry logic
   - MetadataStore: SQLite database for tracking files and deduplication
   - IndexingPipeline: Main orchestrator with:
     * Incremental indexing (SHA-256 dedup)
     * Concurrent async/await API calls
     * Supabase pgvector storage
     * Token usage tracking and statistics
     * Comprehensive error handling and logging

3. search.py (336 lines)
   - Vector similarity search interface
   - VectorSearchClient with:
     * Query embedding generation
     * Supabase pgvector similarity search
     * Fallback direct table queries
     * Configurable similarity thresholds
     * Filtering by source (OneDrive, Google Drive, local)
     * Interactive search session
   - SearchResult dataclass for formatted output
   - Cosine similarity calculation

4. setup_supabase.sql (82 lines)
   - Complete Supabase schema setup
   - pgvector extension with 1536-dimensional embeddings
   - documents table with file metadata and embeddings
   - file_metadata table for dedup tracking
   - match_documents() SQL function for cosine similarity search
   - Optimized indexes (ivfflat for vector similarity, B-tree for metadata)
   - Permission grants for anon and authenticated roles

5. requirements.txt (9 lines)
   Dependencies:
   - openai >= 1.3.0 (Azure OpenAI API)
   - supabase >= 2.0.0 (Postgres + pgvector)
   - tiktoken >= 0.5.0 (Token counting)
   - python-docx >= 0.8.11 (DOCX parsing)
   - pypdf >= 3.0.0 (PDF extraction)
   - aiohttp >= 3.8.0 (Async HTTP)
   - python-dotenv >= 1.0.0 (.env file loading)
   - pydantic >= 2.0.0 (Data validation)

6. .env.example (30 lines)
   - Template for environment configuration
   - Copy to .env and fill in your API keys
   - Includes all configurable parameters

QUICK START
===========

1. Set up Supabase:
   - Create a Supabase project at https://supabase.com
   - Run the SQL from setup_supabase.sql in the SQL editor
   - Copy your project URL and API key

2. Configure Azure OpenAI:
   - Get API key from https://platform.openai.com/api-keys
   - Or use Azure OpenAI Service endpoint if available

3. Install dependencies:
   pip install -r requirements.txt

4. Create .env file:
   cp .env.example .env
   # Edit .env with your API keys and paths

5. Run indexing:
   python indexer.py
   # Scans configured directories, generates embeddings, stores in Supabase

6. Search indexed documents:
   python search.py
   # Interactive search - enter queries and get results with scores

ARCHITECTURE
============

Indexer Flow:
  Directory Scanning → File Hash Check → Text Extraction → Chunking
  → Embedding Generation (Azure) → Rate Limiting → Supabase Storage
  → Metadata Tracking (SQLite)

Search Flow:
  Query Input → Embedding Generation (Azure) → Similarity Search (Supabase)
  → Results Ranking → Formatted Output

Key Classes:
  - DocumentChunker: Configurable text chunking with byte-accurate overlap
  - FileProcessor: Polymorphic file format handling
  - EmbeddingGenerator: Azure API client with rate limiting
  - MetadataStore: SQLite interface for deduplication
  - VectorSearchClient: Supabase query interface
  - IndexingPipeline: Main orchestrator

CONFIGURATION
=============

All settings via environment variables:

Chunking:
  CHUNK_SIZE_BYTES=2048 (target size in bytes)
  CHUNK_OVERLAP_RATIO=0.5 (0.0-1.0)

API Rate Limits:
  MAX_REQUESTS_PER_MINUTE=200
  MAX_TOKENS_PER_MINUTE=150000

Paths (comma-separated for local):
  ONEDRIVE_PATH=/path/to/onedrive
  GOOGLE_DRIVE_PATH=/path/to/google/drive
  LOCAL_PATHS=/path/to/dir1,/path/to/dir2

Supported file extensions are configurable in IndexingConfig.

PERFORMANCE NOTES
=================

- Token Usage: Tracked per-embedding and logged at end
- Rate Limiting: Adaptive delays maintain Azure quota compliance
- Chunk Overlap: 50% overlap creates redundancy for better search context
- Async Concurrency: Uses aiohttp for non-blocking API calls
- Deduplication: SHA-256 hashing prevents duplicate embeddings
- Vector Index: IVFFlat PostgreSQL index optimized for cosine similarity
- Fallback: Direct table query if match_documents() function fails

TROUBLESHOOTING
===============

Invalid API Key:
  - Check AZURE_OPENAI_API_KEY and SUPABASE_API_KEY in .env
  - Verify keys haven't expired in respective dashboards

Rate Limiting (429 errors):
  - Reduce MAX_REQUESTS_PER_MINUTE
  - Check token usage with token counters in logs
  - Azure quota may need plan upgrade

File Type Not Supported:
  - Check supported_extensions tuple in IndexingConfig
  - Extend FileProcessor.extract_text_from_file() for new types

Connection Errors:
  - Verify SUPABASE_URL and AZURE_OPENAI_ENDPOINT
  - Check network connectivity and firewall rules
  - Review application logs for details
